# Contributing to TransferHub

First off, thank you for considering contributing to TransferHub! It's people like you that make TransferHub such a great tool.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issues](../../issues) tab to see if it has already been reported. If not, feel free to open a new issue using the provided templates!

## Setting up for Local Development

1. Fork the repo and create your branch from `main`.
2. Navigate to `/server` and run `npm install`.
3. Navigate to `/client` and run `npm install`.
4. Run tests with `npm test` in both directories to ensure a clean slate.
5. If you've added code that should be tested, add tests.
6. Ensure the test suite passes.
7. Make sure your code follows the existing formatting guidelines (we use Prettier/ESLint).

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
2. Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations and container parameters.
3. Once you open a PR, our CI will automatically run tests and linting.
4. You may merge the Pull Request once you have the sign-off of at least one core developer.

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.
